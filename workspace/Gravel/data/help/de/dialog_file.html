Dateidialog

Abbildung 1: Der Speichern-Dialog am Beispiel GravelML


Der Dateidialog ist der von Java gestellte Dialog des JFileChoosers.
Dieser gliedert sich in ein Eingabefeld für den Dateinamen, ein Auswahlfeld für die letzten besuchten Ordner und eine Ansicht des aktuellen Ordnerinhaltes. In Abbildung 1 ist der Inhalt des Gravel-Ordners in der Version 0.2.1 dargestellt. Unter dieser Auflistung ist ein Filter aktiv. Beim Laden und Speichern von Dateien im GravelML-Format ist als Dateifilter .xml eingestellt, so dass nur Ordner und Dateien mit dem Format .xml hervorgehoben sind.
Darunter sind die Buttons zum Abbrechen und Speichern sowie das Erstellen eines neuen Ordners platziert. Diese Buttons werden in den jeweiligen Dialogen textuell angepasst.

Im Export wird über den Dateifilter das Format gewählt in das exportiert werden soll. Wählt mann dann trotzdem eine ausgegraute Datei, so wird dessen Endung verwendet, um das Export-Format festzustellen. Ebenso gilt dies für eine im Eingabefeld oben spezifizierte Datei mit Endung. Auch diese wird anhand ihrer Endung einem Export-Format zugeordnet.

Wird ein Dateiformat ausgewählt und dann im oberen Eingabefeld ein Dateiname angegeben, der keine Endung hat, so wird die Endung des aktuellen Dateifilters angehängt.
Gibt man also beim Speichern "beispiel" in dem oberen Feld ein, so wird der Graph in der Datei "beispiel.xml" gespeichert.
